通过 [crazyrouter-hermes](https://github.com/xujfcn/crazyrouter-hermes) 可以把 [Hermes Agent](https://github.com/NousResearch/Hermes-Agent) 的模型后端切到 Crazyrouter。配置完成后，Hermes 会使用 Crazyrouter OpenAI 兼容 `base_url`，并通过一个 Crazyrouter API Key 调用 Claude、GPT、DeepSeek、Gemini、Qwen 等 627+ 模型。

## 适合谁用

- 已经安装 Hermes Agent，想直接接入 Crazyrouter 的用户
- 想在本地 Agent 中统一使用 Crazyrouter 模型和额度的用户
- 想用一条命令完成 `OPENAI_API_KEY`、`OPENAI_BASE_URL` 和默认模型配置的用户

## 前置条件

| 项目 | 说明 |
|------|------|
| Hermes Agent | 先在本机安装并确认 Hermes 可启动 |
| Crazyrouter 账号 | 在 [crazyrouter.com](https://crazyrouter.com) 注册 |
| Crazyrouter API Key | 建议为 Hermes 单独创建一个 `sk-...` token |
| 模型权限 | 在 token 里放行你准备在 Hermes 中使用的模型 |

<Tip>
Hermes 使用 OpenAI 兼容配置接入 Crazyrouter，默认接入地址直接使用 `https://cn.crazyrouter.com/v1`。如果你不确定某个客户端到底该填根域名还是 `/v1`，先参考 [API Endpoint 说明](https://docs.crazyrouter.com/en/api-endpoint)。账号登录、充值和控制台仍使用 `https://crazyrouter.com`。
</Tip>

## 手动配置

如果你不想运行脚本，也可以手动写入 Hermes 配置文件。

### 环境变量

编辑 `~/.hermes/.env`：

```bash
OPENAI_API_KEY=sk-your-crazyrouter-key
OPENAI_BASE_URL=https://cn.crazyrouter.com/v1
```

默认保持这个中国接入地址即可。该地址仅用于 API 请求，账号登录、充值和控制台仍使用 `https://crazyrouter.com`：

```bash
OPENAI_BASE_URL=https://cn.crazyrouter.com/v1
```

### 默认模型

编辑 `~/.hermes/config.yaml`：

```yaml
model:
  provider: "custom"
  default: "claude-opus-4-8"
  base_url: "https://cn.crazyrouter.com/v1"
```

默认也建议保持中国接入地址：

```yaml
model:
  provider: "custom"
  default: "claude-opus-4-8"
  base_url: "https://cn.crazyrouter.com/v1"
```

把 `default` 改成你在 Crazyrouter token 中已放行的模型即可。

## 切换模型

配置完成后，可以在 Hermes 内随时用 `/model` 切换模型：

```text
/model claude-opus-4-8
/model gpt-5.5
/model claude-opus-4-8
/model gemini-3.1-pro
/model deepseek-chat
/model qwen-max
/model gpt-4o
```

如果切换后返回 `model not allowed` 或 `403`，通常是当前 Crazyrouter token 没有放行该模型。

## 验证清单

- [ ] `~/.hermes/.env` 中的 `OPENAI_API_KEY` 是有效的 Crazyrouter token
- [ ] `OPENAI_BASE_URL` 是 `https://cn.crazyrouter.com/v1`
- [ ] `~/.hermes/config.yaml` 中 `provider` 为 `custom`
- [ ] 默认模型已在 Crazyrouter token 中放行
- [ ] Hermes 中发送测试消息可以正常返回
- [ ] `/model gpt-5.5` 或其他已放行模型可以切换成功

## 常见问题

| 现象 | 常见原因 | 处理方式 |
|------|----------|----------|
| 401 unauthorized | API Key 写错、过期或被重置 | 重新生成 Crazyrouter token 并更新 `~/.hermes/.env` |
| 403 / model not allowed | token 未放行当前模型 | 在 Crazyrouter token 设置中放行对应模型 |
| 仍然走旧服务商 | Hermes 未读取新配置，或配置文件路径不对 | 重启 Hermes，并检查 `~/.hermes/.env`、`~/.hermes/config.yaml` |
| 连接超时 | 本机网络无法访问 Crazyrouter | 检查网络代理、防火墙和 DNS |
