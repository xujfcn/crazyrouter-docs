# Gemini 图片编辑

```
POST /v1beta/models/{model}:generateContent
```

截至 2026 年 4 月 14 日，Crazyrouter 对 Gemini 图片编辑相关路径的最新复测结论如下：

- `gemini-3.1-flash-image-preview` 原生 `generateContent` 已重新验证可返回图片
- `gemini-2.5-flash-image` 原生 `generateContent` 也能返回图片，但计费一致性仍在复核中
- Nano Banana family 走 `POST /v1/images/generations` 的公开 Images 兼容路径当前不健康

当前不健康的是旧的公开 Images 兼容链路，而不是全部 Gemini 原生图片路径。对图片编辑或参考图工作负载，当前应优先收敛到 Gemini 原生 `generateContent`。

<Warning>
  当前不要再把 Nano Banana family 的 `POST /v1/images/generations` 写成稳定图片编辑入口。`nano-banana-2` 应优先改用 `gemini-3.1-flash-image-preview:generateContent`；`nano-banana` 仍只建议受控测试。
</Warning>

<Note>
  当前更可用的路径是：
  [Nano Banana 2](/images/nano-banana-2) 对应的 `gemini-3.1-flash-image-preview:generateContent`。
  [Nano Banana](/images/nano-banana) 当前只保留 `gemini-2.5-flash-image:generateContent` 的受控测试说明，不建议直接写成稳定生产能力。
</Note>

---

## 当前建议

1. 如果你的目标是 Gemini 文生图，优先参考 [Gemini 图片生成](/chat/gemini/image-gen)
2. 如果你的目标是当前可接入的参考图编辑，优先参考 [Nano Banana 2](/images/nano-banana-2)
3. 如果你必须评估 `nano-banana`，只按 [Nano Banana](/images/nano-banana) 页的受控测试方式验证
4. 不要再把 `POST /v1/images/generations` 当成 Nano Banana family 的默认入口

---

## Retest 模板

下面是当前更推荐的最小请求模板。它表示“如何用 Gemini 原生路径重测或接入”，不再推荐把相同需求写成 `/v1/images/generations` 教程：

```bash
curl "https://crazyrouter.com/v1beta/models/gemini-3.1-flash-image-preview:generateContent?key=YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "contents": [
      {
        "role": "user",
        "parts": [
          {"text": "Edit this image: change the background to blue and keep the main square visible. Return image output."},
          {
            "inlineData": {
              "mimeType": "image/png",
              "data": "iVBORw0KGgoAAA..."
            }
          }
        ]
      }
    ],
    "generationConfig": {
      "responseModalities": ["IMAGE"]
    }
  }'
```
