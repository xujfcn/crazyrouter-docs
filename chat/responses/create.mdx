# 创建模型响应

```
POST /v1/responses
```

截至 2026 年 3 月 23 日，Crazyrouter 生产环境已验证：

- `gpt-5.4` 可以正常返回 `object: "response"`
- 基础文本请求的 `output` 当前可稳定拿到 `message`
- 流式请求可稳定观察到 `response.created`、`response.in_progress`、`response.output_item.added`、`response.content_part.added`、`response.output_text.delta`

---

## 基本请求

```bash
curl https://crazyrouter.com/v1/responses \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-5.4",
    "input": "用一句话解释什么是 API"
  }'
```

生产环境实际响应的关键形态：

```json
{
  "object": "response",
  "model": "gpt-5.4",
  "output": [
    {
      "type": "message"
    }
  ]
}
```

---

## 流式请求

```bash
curl https://crazyrouter.com/v1/responses \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-5.4",
    "input": "Say hello in one sentence.",
    "stream": true
  }'
```

2026-03-23 的生产环境流式探针观察到的事件顺序开头如下：

```text
event: response.created
event: response.in_progress
event: response.output_item.added
event: response.content_part.added
event: response.output_text.delta
```

因此当前最稳妥的处理方式是：

1. 用 `response.output_text.delta` 逐步拼接文本
2. 用 `response.completed` 或连接结束作为收尾信号
