# 视频生成别名

```
POST /v1/video/generations
```

这是 Crazyrouter 提供的兼容别名路由，适合那些固定使用 `/v1/video/generations` 路径的视频客户端或中间层。

## 认证

```http
Authorization: Bearer YOUR_API_KEY
```

## 什么时候用它

- 你的客户端已经写死了 `/v1/video/generations`
- 你在迁移历史视频工作流，不想改调用路径
- 你需要兼容第三方已经发布的请求格式
- 你要提交 Wan 文生视频，例如 `wan2.5-t2v-preview` 或 `wan2.2-t2v-plus`

如果你在开发新接入，优先考虑：

- Wan 文生视频：使用本页的 `POST /v1/video/generations`
- Veo / 统一视频工作流：使用 `POST /v1/video/create`，并提交 `veo-3.1-fast` 或 `veo-3.1-quality`
- OpenAI 官方兼容视频工作流：使用 `POST /v1/videos`

## 最小请求示例

### Wan 2.2 T2V Plus

```bash cURL
curl -X POST https://api.crazyrouter.com/v1/video/generations \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "wan2.2-t2v-plus",
    "prompt": "一段平静的电影感山间云海镜头",
    "seconds": "5",
    "size": "480P"
  }'
```

### 通用最小示例

```bash cURL
curl -X POST https://api.crazyrouter.com/v1/video/generations   -H "Content-Type: application/json"   -H "Authorization: Bearer YOUR_API_KEY"   -d '{
    "model": "wan2.5-t2v-preview",
    "prompt": "A cinematic shot of waves hitting a cliff at sunset"
  }'
```

## 典型成功响应

```json
{
  "id": "video_task_abc123",
  "task_id": "video_task_abc123",
  "status": "processing",
  "status_update_time": 1709123456
}
```

## 本地验证到的错误响应

在本地 `http://127.0.0.1:4000` 环境中，这个路由已经验证存在。当模型临时不可用时，返回的是业务错误而不是 `404`：

```json
{
  "code": "get_channel_failed",
  "message": "model wan2.2-t2v-plus is temporarily unavailable, please try again later",
  "data": null
}
```

## 注意事项

- 请求体字段会随具体视频模型和上游能力而变化
- `model` 是否可用取决于当前 Token 白名单和上游渠道状态
- Wan 文生视频使用 `model`、`prompt`、`seconds`、`size`；`wan2.2-t2v-plus` 的已验证组合是 `seconds: "5"`，`size: "480P"` 或 `"1080P"`。
- `wan2.2-t2v-plus` 不应提交到 `/alibailian/api/v1/services/aigc/video-generation/video-synthesis`。
- 任务创建成功后，可继续调用 `GET /v1/video/generations/{task_id}` 查询状态

<Note>
这个路由是兼容别名，不代表所有视频模型都共享完全一致的请求字段。Wan 文生视频当前推荐使用本页路由；Veo 或统一视频工作流应使用 `/v1/video/create`，并提交 `veo-3.1-fast` 或 `veo-3.1-quality`。
</Note>
