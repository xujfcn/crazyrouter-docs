# Nano Banana 2

```
当前推荐路径
POST /v1beta/models/gemini-3.1-flash-image-preview:generateContent
```

截至 `2026-04-14`，Crazyrouter 对 `nano-banana-2` 的最新生产复测结论如下：

- `nano-banana-2` 在服务端当前映射到 `gemini-3.1-flash-image-preview`
- 当前推荐直接调用 Gemini 原生 `generateContent`
- 当前不再推荐通过 `POST /v1/images/generations` 调用 `nano-banana-2`
- 同日生产复测中，原生路径请求成功，计费看起来正常

<Warning>
  不要再把 `nano-banana-2` 写成 `POST /v1/images/generations` 的稳定推荐模型。当前推荐入口已经改为 Gemini 原生 `POST /v1beta/models/gemini-3.1-flash-image-preview:generateContent`。
</Warning>

## 当前风险

### 1. 公开 Images 兼容路径当前不健康

`nano-banana-2` 走 `POST /v1/images/generations` 时，`2026-04-14` 的生产与本地 `4000` 复测都出现了以下错误簇：

- `Unknown name "imageOutputOptions" at 'generation_config.image_config'`
- `not supported model for image generation`
- `get_channel_failed`

因此当前不应再把 `/v1/images/generations` 写成它的公开稳定入口。

### 2. 原生响应仍需按 Gemini 形态解析

虽然当前推荐走原生路径，但客户端仍要按 Gemini 原生响应处理：

1. 优先读取 `candidates[].content.parts[].inlineData`
2. 不要假设它返回 OpenAI Images 风格的 `data[].url`
3. 如果后续扩到其他 Gemini 图片模型，还要兼容 `text` 中的 `data:image/...` 或外链 URL

## 当前建议

1. 生产接入 `nano-banana-2` 时，直接改用 `gemini-3.1-flash-image-preview:generateContent`
2. 不要再使用 `POST /v1/images/generations` + `model: "nano-banana-2"`
3. 如果你要做自动化取图，优先按 `inlineData` 解析

## 推荐请求模板

```bash
curl "https://crazyrouter.com/v1beta/models/gemini-3.1-flash-image-preview:generateContent?key=YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "contents": [
      {
        "role": "user",
        "parts": [
          {
            "text": "Generate an IMAGE of a clean ecommerce hero shot on a white background. Return image output."
          }
        ]
      }
    ],
    "generationConfig": {
      "responseModalities": ["IMAGE"]
    }
  }'
```

## 旧路径说明

以下旧写法当前已不再推荐：

```bash
curl -X POST https://crazyrouter.com/v1/images/generations \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "nano-banana-2",
    "prompt": "..."
  }'
```

这条路径当前的问题不是单次抖动，而是生产与本地 `4000` 复测都可复现。
