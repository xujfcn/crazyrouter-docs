## 本轮几项关键增强内容

### Coze

已经将公开主线调整为更稳妥的方案：

- API 插件
- 工作流 HTTP 请求节点

同时明确不再默认把“企业版自定义模型接入”写成公开默认路径，避免误导普通用户。

### ChatBox

已经补成完整桌面端教程，明确了：

- 使用 `OpenAI API` / `OpenAI API Compatible`
- `API Host` 使用 `https://crazyrouter.com`
- `API Path` 保持默认或留空
- 首轮先用 `gpt-4.1`
- 知识库 / 文件问答应在最小链路验证成功后再启用

### Zotero

已经明确区分两类插件：

- 支持自定义 `Base API URL / API URL / Endpoint` 的插件：可以作为 Crazyrouter 公开接入方案
- 只支持填写官方 `OpenAI API Key` 的插件：不适合作为公开默认文档方案

这避免了“任意 Zotero AI 插件都能接 Crazyrouter”的误导。

## 公开文档边界已经理顺

已经明确不再把以下内容作为普通用户公开文档主线：

- `/api/status/page`
- `/api/status/health`
- `/healthz/live`
- `/healthz/ready`
- `/api/user/self`

原因分别是：

- 前端页面数据接口
- 运维健康检查接口
- 部署探针
- 控制台 / 内部账户信息接口

同时，已将真正适合普通用户查看的状态页引导到：

- `https://crazyrouter.com/status`

## Token 管理文档也已重新定位

已经将 `/api/token/*` 相关文档重新定位为：

- API Key / 控制台管理用途

而不是主模型调用接口，避免用户误以为这些是普通 `sk-xxx` 模型调用 API。
