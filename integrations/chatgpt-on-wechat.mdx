## 概述

ChatGPT on WeChat 是一个常见的微信机器人项目，可以把 Crazyrouter 的模型能力接到私聊和群聊场景。对接时建议继续使用它默认的 OpenAI 兼容配置方式，并先用一个已实测成功的模型跑通最小回复链路。

## 推荐首轮配置

- `open_ai_api_base`: `https://crazyrouter.com/v1`
- 首个验证模型：`gpt-5.4`
- 首轮只开放少量模型，避免排障范围过大

## 配置方法

### 修改配置文件

编辑 `config.json`：

```json
{
  "open_ai_api_key": "sk-xxx",
  "open_ai_api_base": "https://crazyrouter.com/v1",
  "model": "gpt-5.4",
  "proxy": "",
  "single_chat_prefix": [""],
  "single_chat_reply_prefix": "",
  "group_chat_prefix": ["@bot"],
  "group_name_white_list": ["AI交流群"]
}
```

### 环境变量方式

```bash
export OPEN_AI_API_KEY=sk-xxx
export OPEN_AI_API_BASE=https://crazyrouter.com/v1
export MODEL=gpt-5.4
```

### Docker 部署

```yaml
services:
  chatgpt-on-wechat:
    image: zhayujie/chatgpt-on-wechat
    environment:
      - OPEN_AI_API_KEY=sk-xxx
      - OPEN_AI_API_BASE=https://crazyrouter.com/v1
      - MODEL=gpt-5.4
    volumes:
      - ./config.json:/app/config.json
```

## 推荐模型

| 场景 | 模型 | 说明 |
|------|------|------|
| 首轮链路验证 | `gpt-5.4` | 2026 年 3 月 23 日已在生产环境实测成功，最适合先确认 OpenAI 兼容链路 |
| 高质量长回复 | `claude-sonnet-4-6` | 更适合复杂解释、总结和长文本 |
| Gemini 备用档 | `gemini-3-pro-preview` | 适合补充第二条供应商兼容性验证路径 |

## 建议验证顺序

1. 先只配置一个 token
2. 先只配置一个模型 `gpt-5.4`
3. 先在私聊里发送 `Reply only OK`
4. 确认 Crazyrouter 后台日志已有对应请求
5. 再逐步放开群聊、更多前缀规则和其他模型

<Warning>
  微信机器人项目需要遵守微信使用规范，避免频繁发送消息导致账号受限。
</Warning>
