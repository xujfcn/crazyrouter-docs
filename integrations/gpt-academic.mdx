## 概述

GPT Academic（学术版 GPT）面向论文润色、翻译、代码解释等场景。对接 Crazyrouter 时，建议继续使用它的 OpenAI URL 重定向方式，但默认模型要切换到当前已在生产环境验证过的最新可用模型。

## 配置方法

### 修改配置文件

编辑 `config.py` 或 `config_private.py`：

```python
API_KEY = "sk-xxx"
API_URL_REDIRECT = {
    "https://api.openai.com/v1/chat/completions": "https://crazyrouter.com/v1/chat/completions"
}

# 或者直接设置
LLM_MODEL = "gpt-5.4"
AVAIL_LLM_MODELS = [
    "gpt-5.4",
    "claude-sonnet-4-6",
    "gemini-3-pro-preview"
]
```

### 环境变量方式

```bash
export API_KEY=sk-xxx
export API_URL_REDIRECT='{"https://api.openai.com/v1/chat/completions": "https://crazyrouter.com/v1/chat/completions"}'
export LLM_MODEL=gpt-5.4
```

### Docker 部署

```yaml
services:
  gpt-academic:
    image: ghcr.io/binary-husky/gpt_academic
    ports:
      - "22303:22303"
    environment:
      - API_KEY=sk-xxx
      - LLM_MODEL=gpt-5.4
      - API_URL_REDIRECT={"https://api.openai.com/v1/chat/completions": "https://crazyrouter.com/v1/chat/completions"}
```

## 推荐模型

| 场景 | 模型 | 说明 |
|------|------|------|
| 首轮学术助手验证 | `gpt-5.4` | 2026 年 3 月 23 日已在生产环境实测成功，最适合作为 OpenAI 兼容基线 |
| 高质量长文总结 | `claude-sonnet-4-6` | 更适合复杂解释、长文重写和总结 |
| Gemini 备用档 | `gemini-3-pro-preview` | 适合补充第二条供应商兼容性验证路径 |

<Note>
  GPT Academic 的 URL 重定向能力可以把 OpenAI 请求直接转到 Crazyrouter。建议先只用 `gpt-5.4` 跑通最小会话，再逐步增加模型列表和复杂插件能力。
</Note>
