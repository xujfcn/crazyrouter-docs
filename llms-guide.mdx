# 把文档交给 AI 工具

如果你正在用 ChatGPT、Claude、Cursor、Cline、Aider、Codex 等 AI 工具写代码，可以直接把下面这个文档入口发给它：

```text
https://docs.crazyrouter.com/llms.txt
```

AI 工具读取后，可以快速找到 Crazyrouter 的快速开始、认证方式、API 端点、模型列表、图像/视频/音频接口和常见工具接入文档。

## 推荐复制

把这段提示词复制给 AI 工具：

```text
请先阅读 https://docs.crazyrouter.com/llms.txt，然后基于 Crazyrouter 官方文档回答我的问题。

我的需求是：[在这里写你的需求，例如：用 Python 调用 Sora 2 生成 5 秒视频]

请注意：
- OpenAI 兼容 SDK 默认使用 base_url：https://api.crazyrouter.com/v1
- 如果涉及模型名称、价格或计费方式，请以 https://crazyrouter.com/pricing 为准
- 如果不确定某个模型是否可用，请提醒我到 Pricing 页面或 /v1/models 确认
- 请给出可以直接运行的代码示例
```

## 常见用法

| 你想让 AI 做什么 | 推荐说法 |
| --- | --- |
| 生成接入代码 | “阅读 llms.txt，然后用 Python/Node.js 写一个 Crazyrouter 调用示例。” |
| 接入某个模型 | “阅读 llms.txt，并确认这个模型应该使用哪个 Crazyrouter 接口。” |
| 配置 IDE 或 Agent | “阅读 llms.txt，然后告诉我如何在 Cursor / Cline / Claude Code 中配置 Crazyrouter。” |
| 排查报错 | “阅读 llms.txt 和错误处理文档，帮我分析这个 Crazyrouter API 报错。” |
| 生成视频或图片 | “阅读 llms.txt，然后按 Crazyrouter 文档给出视频/图片生成请求示例。” |

## 什么时候用 llms-full.txt

`llms.txt` 是精简索引，适合 AI 先判断应该读取哪些文档。

如果 AI 工具支持读取更长文本，或者你希望它一次性获得完整文档上下文，可以继续提供：

```text
https://docs.crazyrouter.com/llms-full.txt
```

## 模型和价格以哪里为准

模型会随渠道、权限和平台配置变化。文档示例只用于说明调用方式，不代表完整模型清单。

查看当前公开模型、价格和计费方式，请使用：

```text
https://crazyrouter.com/pricing
```

如果你已经有 API Key，并想确认当前 Token 实际可调用哪些模型，可以请求：

```bash cURL
curl https://api.crazyrouter.com/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"
```

<Warning>
不要把真实 API Key 粘贴到不可信的 AI 工具或公开对话中。需要让 AI 帮你写代码时，使用 `YOUR_API_KEY` 这样的占位符即可。
</Warning>

## 单页 Markdown

很多文档页面也可以用 Markdown 形式读取。把页面 URL 后面加上 `.md` 即可，例如：

```text
https://docs.crazyrouter.com/quickstart.md
https://docs.crazyrouter.com/api-endpoint.md
https://docs.crazyrouter.com/video/sora.md
```

如果 AI 工具不能很好地读取整个网站，可以先给它 `llms.txt`，再按它需要的页面继续提供对应 `.md` 链接。
