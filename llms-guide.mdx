# 如何使用 llms.txt

`https://docs.crazyrouter.com/llms.txt` 是给 Codex、Claude Code、ChatGPT、Cursor、Cline、Aider 等 AI 工具读取的 Crazyrouter 文档入口。用户只需要把这个 URL 发给 AI 工具，AI 就可以先读取精简索引，再按任务继续读取对应文档页面。

## 推荐给 AI 的提示词

```text
请先读取 https://docs.crazyrouter.com/llms.txt，并基于其中的 Crazyrouter 文档索引回答我的问题。

要求：
1. 模型名称、价格、计费方式和端点支持必须以 https://crazyrouter.com/pricing 以及 https://crazyrouter.com/api/pricing?lang=zh 暴露的数据为准。
2. 如果需要确认当前 API Key 实际可用模型，请调用 GET https://cn.crazyrouter.com/v1/models，并使用 Authorization: Bearer YOUR_API_KEY。
3. 不要根据旧知识猜测模型名、价格或端点；如果 pricing 或 /v1/models 中没有该模型，请说明需要先确认模型是否已开放。
4. 调用方法必须与 Crazyrouter 文档和后端路由保持一致；OpenAI 兼容客户端使用 base_url https://cn.crazyrouter.com/v1，Claude / Anthropic 原生客户端使用根域名 https://cn.crazyrouter.com。
5. 生成代码前，先根据 /api/pricing 返回的 public_endpoint_types 或 supported_endpoint_types 选择正确接口。
```

## 实时事实源

| 需要确认什么 | 使用哪个来源 |
| --- | --- |
| 公开模型、价格、计费方式、端点类型 | [Pricing 页面](https://crazyrouter.com/pricing) 和 `GET https://crazyrouter.com/api/pricing?lang=zh` |
| 当前 API Key 可用模型 | `GET https://cn.crazyrouter.com/v1/models` |
| 文档索引 | `https://docs.crazyrouter.com/llms.txt` |
| 完整文档上下文 | `https://docs.crazyrouter.com/llms-full.txt` |
| 单页 Markdown | 在文档 URL 后追加 `.md`，例如 `https://docs.crazyrouter.com/quickstart.md` |

<Warning>
不要把 `llms.txt` 当成全量模型清单。模型会随渠道、分组、Token 权限和定价配置变化；全量模型与价格必须以 Pricing 页面和 `/api/pricing` 为准。
</Warning>

## 端点选择规则

`GET https://crazyrouter.com/api/pricing?lang=zh` 返回的每个模型条目可能包含：

- `model_name`：调用 API 时使用的公开模型名
- `public_endpoint_types`：面向用户展示的推荐端点类型，优先使用
- `supported_endpoint_types`：后端支持的端点类型
- `video_pricing` / `image_pricing`：视频或图像模型的计费细则
- `billing_mode` / `display_unit`：计费口径与展示单位

常见端点类型和调用路径：

| endpoint type | 调用路径 |
| --- | --- |
| `openai` | `POST /v1/chat/completions` |
| `openai-response` | `POST /v1/responses` |
| `anthropic` | `POST /v1/messages` |
| `gemini` | `POST /v1beta/models/{model}:generateContent` |
| `image-generation` | `POST /v1/images/generations` 或 `POST /v1/images/edits` |
| `openai-video` | `POST /v1/video/generations` 或 `POST /v1/videos` |
| `unified-video` | `POST /v1/video/create`，查询用 `GET /v1/video/query` |
| `kling` | `/kling/v1/videos/*` 或 `/kling/v1/images/*` |
| `minimax-video` | `/minimax/v1/video_generation` |
| `luma` | `/luma/generations` |
| `runway` | `/runwayml/v1/image_to_video` |
| `volc-video` | `/volc/v1/contents/generations/tasks` |
| `ali-video` | `/alibailian/api/v1/services/aigc/video-generation/video-synthesis` |

<Note>
端点映射与本地仓库中的 `router/relay-router.go`、`router/video-router.go`、`controller/pricing.go`、`controller/model.go`、`common/endpoint_type.go` 保持一致。文档维护时应先核对这些文件，再更新示例。
</Note>

## 用户应该怎么引用

在 AI 工具里直接贴：

```text
请阅读 https://docs.crazyrouter.com/llms.txt，然后告诉我如何用 Crazyrouter 调用 [你的任务]。请先确认模型和端点是否与 pricing 页面一致。
```

如果任务需要完整上下文，再追加：

```text
如果 llms.txt 中的信息不足，请继续读取 https://docs.crazyrouter.com/llms-full.txt 和相关单页 Markdown。
```

## 维护要求

更新 `crazyrouter-docs/llms.txt` 时遵守以下规则：

1. 不在 `llms.txt` 中硬编码全量模型列表。
2. 示例模型必须能在 Pricing 页面或 `/api/pricing` 中找到。
3. 示例调用路径必须能在文档和本地 Go 路由中找到。
4. 模型可用性说明必须提醒用户以 `/v1/models` 的 Token 级结果为准。
5. Mintlify 文档部署后，用户继续引用同一个 URL：`https://docs.crazyrouter.com/llms.txt`。
